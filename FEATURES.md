# PENTACODE — Cryptographic Security Objectives

This document outlines the theoretical security advantages and architectural mechanics implemented in **Pentacode V.3** to protect encrypted payloads against contemporary cryptanalysis techniques.

---

### 1. Vector Normalization (MAP1 Mapping)
* **Cryptographic Function:** Normalizes the plaintext coordinate space into strict mathematical vectors $\mathbb{Z}_{[1, 69]}$.
* **Security Mitigation:** Structural formatting vulnerabilities, character case indicators, spatial markers, and punctuation are completely flattened. Converting raw input elements into unified numeric entities ensures that mathematical and cryptographic modular arithmetic functions can be applied uniformly across the entire payload domain.

### 2. Private Key-Seeded Permutation Alphabet (MAP2)
* **Cryptographic Function:** Prevents frequency analysis attacks by eliminating globally static character definitions.
* **Security Mitigation:** Standard substitution ciphers map explicit coordinates to a static, public glyph set. Pentacode maps execution coordinates to an array of **205 unique cryptographic symbols** whose indexes are dynamically randomized using a high-entropy Fisher-Yates shuffle seeded via a 256-bit SHA-256 byte stream. Knowing the underlying algorithm provides zero utility to an attacker; without the specific layout topology generated by the unique private key, creating a global decryption lookup table is mathematically impossible.

### 3. Dynamic Public Key Initialization Vectors
* **Cryptographic Function:** Destroys ciphertext determinism and ensures semantic security.
* **Security Mitigation:** Re-encrypting an identical plaintext string with a fixed symmetric key usually produces an identical ciphertext signature. A **20-character random public key** (first 20 chars of output) is generated using `crypto.getRandomValues` and layered into the stream calculation. This guarantees that identical source texts processed with the same private key will output entirely distinct, non-repeating ciphertext arrays on every execution.

### 4. Polyalphabetic Vector Blending (Multi-Stream Aggregation)
* **Cryptographic Function:** Flattens language structures and frequency distributions.
* **Security Mitigation:** The core math combines separate dimensions by calculating the sum of the Plaintext coordinate, the repeating Private Key coordinate, and the repeating Public Key coordinate ($P_i + K_i + V_i$). Repeating characters in the source text yield radically divergent sums due to shifting key alignments.

### 5. Private Key-Driven Length Stream Variance
* **Cryptographic Function:** Shatters character alignment indexes, ciphertext parsing, and structural signature tracking.
* **Security Mitigation:** Derives an unpredictable layout stream from `privateKey + "_lengths"` (via chained SHA-256) to dynamically expand each plaintext character into **1 or 2 ciphertext symbols**. This throws off geometric alignment and masks individual character boundaries.

### 6. Sub-Sum Code Step Expansion
* **Cryptographic Function:** Obfuscates multi-character block footprints and prevents visual or algorithmic data clustering.
* **Security Mitigation:** When expanding to 2 symbols, secondary sub-symbols are incremented linearly ($\text{BaseSum} + s$). This makes variable-length blocks blend seamlessly into the ciphertext stream.

### 7. Multi-Round Grid Transposition Cipher (New in V.3)
* **Cryptographic Function:** Applies **5 rounds** of dynamic transposition before substitution.
* **Security Mitigation:** 
  - Packs plaintext into a grid whose width is derived from key length (`gridDims`).
  - Performs **multiplicative row shifts** based on key length.
  - Applies **row and column permutations** derived from sorted ASCII order of the (evolving) key.
  - Reads out via **anti-diagonal traversal**.
  - Evolves the key each round via `deriveSubkey` (cyclic character reordering based on ASCII modulo length).
  
  This creates a highly dynamic, key-dependent transposition layer that thoroughly scrambles positional relationships before the polyalphabetic substitution stage.

---

**Implementation Notes:**
- All randomness (MAP1/MAP2 shuffles, length stream, public key) is **deterministically derived** from the private key using chained SHA-256, except for the public initialization vector.
- Supports a rich character set including letters, digits, spaces, newlines, and common punctuation.
- Built as a self-contained browser-based tool using Web Crypto API.
