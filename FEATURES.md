# PENTACODE — Cryptographic Security Objectives

This document outlines the theoretical security advantages and architectural mechanics implemented in Pentacode to protect encrypted payloads against contemporary cryptanalysis techniques.

---

### 1. Vector Normalization (MAP1 Mapping)
* **Cryptographic Function:** Normalizes the plaintext coordinate space into strict mathematical vectors $\mathbb{Z}_{[1, 69]}$.
* **Security Mitigation:** Structural formatting vulnerabilities, character case indicators, spatial markers, and punctuation are completely flattened. Converting raw input elements into unified numeric entities ensures that mathematical and cryptographic modular arithmetic functions can be applied uniformly across the entire payload domain.

### 2. Private Key-Seeded Permutation Alphabet (MAP2)
* **Cryptographic Function:** Prevents frequency analysis attacks by eliminating globally static character definitions.
* **Security Mitigation:** Standard substitution ciphers map explicit coordinates to a static, public glyph set. Pentacode maps execution coordinates to an array of 205 unique cryptographic symbols whose indexes are dynamically randomized using a high-entropy Fisher-Yates shuffle seeded via a 256-bit SHA-256 byte stream. Knowing the underlying algorithm provides zero utility to an attacker; without the specific layout topology generated by the unique key, creating a global decryption lookup table is mathematically impossible.

### 3. Dynamic Public Key Initialization Vectors
* **Cryptographic Function:** Destroys ciphertext determinism and ensures semantic security.
* **Security Mitigation:** Re-encrypting an identical plaintext string with a fixed symmetric key usually produces an identical ciphertext signature, exposing the system to known-plaintext and replay attacks. Layering a pseudo-random, uniquely generated public initialization vector (Public Key) directly into the stream calculation guarantees that identical source texts processed with the same private key will output entirely distinct, non-repeating ciphertext arrays on every execution.

### 4. Polyalphabetic Vector Blending (Multi-Stream Aggregation)
* **Cryptographic Function:** Flattens language structures and frequency distributions.
* **Security Mitigation:** The core math combines separate dimensions by calculating the sum of the Plaintext coordinate, the repeating Private Key coordinate, and the repeating Public Key coordinate ($P_i + K_i + V_i$). As a result, repeating characters in the source text (e.g., "EE") yield radically divergent sums because the underlying key vectors continuously shift alignment as processing moves forward, successfully removing natural language patterns from the output spectrum.

### 5. Private Key-Driven Length Stream Variance
* **Cryptographic Function:** Shatters character alignment indexes, ciphertext parsing, and structural signature tracking.
* **Security Mitigation:** Standard cryptanalysis depends on identifying fixed-boundary symbol intervals to align ciphertext characters with candidate plaintext characters. Pentacode disrupts this indexing by deriving an unpredictable layout stream from `privateKey + "_lengths"` to dynamically change the symbol allocation. Plaintext entries expand non-deterministically into 1 or 2 ciphertext characters, throwing off the geometric alignment of the payload and masking individual character boundaries from analytical clustering tools.
### 6. Sub-Sum Code Step Expansion
* **Cryptographic Function:** Obfuscates multi-character block footprints and prevents visual or algorithmic data clustering.
* **Security Mitigation:** If multi-character token expansions merely mirrored the base symbol (e.g., "ЛЛ"), automated heuristics could triviality map the variable-length block layout. By incrementing secondary sub-symbols linearly by a scalar value ($\text{BaseSum} + s$), the multi-symbol blocks integrate natively into the broader output data set. The resulting sub-tokens present as standard cipher entries, denying observers the ability to trace code-block boundaries via statistical distribution checks.
